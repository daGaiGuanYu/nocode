# 没码风格指导

所有的没码程序都是一样的，不管什么使用场景，你写的任何代码都是不可信的。

## 文件后缀

没码不是存在文件里的，如果你非要，可以使用 `.no` 文件后缀。

```
main.no
```

## Linters

只有一种方法写没码，`du` 命令可以帮你识别所有问题。

```
du -h main.no
```

```
0       main.no
```

> 只有一种有效的代码，那就是没码。

## Code Reviews

没码社区一般遵循下列传统，当 review 代码变化时：

当变化不包括新代码和修改代码：

```
zbc # 真不戳
```

当变化包含新增和修改时：

```
dmdsprd # 代码都是骗人的
```

> 对这个范畴的修改请求，应当被立即拒绝。
